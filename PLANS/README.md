> **חוק.** כל תוכנית **נושאית** שסוכן כותב לפרויקט הזה — חייבת להישמר כקובץ
> Markdown בתוך התיקייה הזו, בקטגוריה המתאימה. אסור להשאיר תוכניות בשורש

## 1. מתי לכתוב תוכנית

כתוב תוכנית **לפני** שאתה מתחיל לבצע משימה שעונה לאחד מהקריטריונים:

- נוגעת ביותר מקובץ אחד.
- נוגעת בסכימה / ב-Constitution / באינווריאנט ארכיטקטוני.
- כוללת יותר מ-3 צעדים.
- משנה התנהגות UI/UX שהמשתמש יראה.
- דורשת אישור אסתטי / תוכני מהמשתמש.

## 2. מבנה הקטגוריות

```
PLANS/
├── README.md                ← הקובץ הזה (חוקים)
├── next-steps/              ← תוכניות עתידיות פעילות (לסגור לפני launch)

├── features/                ← פיצ'רים חדשים (יווצר לפי צורך)
├── design/                  ← שינויי עיצוב, טיפו, פלטה, motion (יווצר לפי צורך)

├── performance/             ← LCP, CLS, אופטימיזציה (יווצר לפי צורך)
```

## 3. כללי שמות קבצים

פורמט: `YYYY-MM-DD_kebab-case-title.md`

דוגמאות תקינות:
- `2026-05-28_hero-headline-copy-revision.md`
- `2026-05-28_lcp-image-optimization.md`

- תאריך = יום יצירת התוכנית.
- כותרת = תיאור קצר וברור באנגלית, kebab-case.
- אסור רווחים, אסור עברית בשם הקובץ (תוכן הקובץ עצמו — בעברית מותר).

---

## 4. תבנית תוכנית (חובה)

```markdown
# [כותרת התוכנית]

**Status:** draft | approved | in-progress | done | archived
**Category:** fixes | features | refactors | design | performance | research
**Created:** YYYY-MM-DD
**Author agent:** [שם הסוכן]
**Related:** [קישורים לתוכניות / קבצים / סקציות ב-Constitution]

---

## 1. הקשר ומטרה
מה המצב היום? מה הבעיה / ההזדמנות?

## 2. תוצאה רצויה (Definition of Done)
איך נדע שהתוכנית הושלמה? קריטריונים מדידים.

## 3. אילוצים מה-Constitution
אילו סעיפים ב-`CLAUDE.md` רלוונטיים? מה אסור לשבור?

## 4. צעדים
- [ ] צעד 1 — קובץ ספציפי, שינוי ספציפי
- [ ] צעד 2 — ...

## 5. סיכוני עיצוב / רגרסיות
מה יכול להישבר? איך נבדוק?

## 6. אישור משתמש נדרש?
כן/לא — ועל מה ספציפית.
```

---

## 5. מחזור חיים של תוכנית

1. **draft** — סוכן כתב, מחכה לקריאה / אישור.
2. **approved** — אושר. אפשר להתחיל.
3. **in-progress** — בעבודה. עדכן צ'קבוקסים תוך כדי.
4. **done** — DoD הושג. הוסף שורת "Completed: YYYY-MM-DD".
5. **archived** — בוצעה / נזנחה. **העבר ל-`archive/`** (אל תמחק).

---

## 6. עקרונות

- **תוכנית לפני קוד.** סוכן שמתחיל בלי לכתוב תוכנית במקרים שדורשים — מפר פרוטוקול.
- **קובץ אחד = תוכנית אחת.**
- **קצר, ספציפי, מדיד.** 50–200 שורות. אם > 300, פצל.
- **קישורים, לא העתקה.** הפנה ל-`CLAUDE.md` / `architecture/*`.
- **שפה: עברית** לתוכן, **אנגלית** לשם הקובץ ולמונחים טכניים.
- **לא למחוק.** תוכניות הן זיכרון של הפרויקט.
